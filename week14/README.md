# 🎓 Week 14：期末整合專題 — 成果發表與系統展示

**雲端與數位內容管理 ｜ 國立金門大學 工業工程與管理學系**

> **本週課程理念：** 這不是一堂課，而是一場發表會。
>
> 十三週前，你可能從來沒有用過 Make、串接過 API、

## 📊 報告結構建議（10 分鐘）

### 第一段：問題背景（2 分鐘）

說明你們選擇的情境和它解決的真實問題：

**引導問題：**
- 在沒有這套系統之前，這件事是怎麼做的？
- 這樣做有什麼痛點？（時間、人力、錯誤率）
- 你們的系統讓什麼事情變得更好？

**建議的呈現方式：**
````
Before（沒有系統時）：
  人工查職缺：每週 3 小時
  手動整理資料：每週 2 小時
  忘記截止日：每學期 2-3 次

After（有了系統後）：
  自動採集 + AI 分析：0 人時
  資料自動整理入庫：0 人時
  截止日自動提醒：0 漏失
````

---

### 第二段：系統架構說明（3 分鐘）

展示你們的系統是怎麼設計的：

**必須包含：**
1. **工具地圖**：用哪些工具，每個工具的角色
2. **資料流程圖**：資料從哪裡進來，經過哪些處理，輸出什麼

**建議展示方式：**
- 開啟 GitHub 展示網站的系統架構區塊
- 或在 Notion 頁面展示架構圖
````
展示腳本範例：
「我們的系統共分四層：
 第一層是資料採集——ParseHub 每週一自動爬取...
 第二層是 AI 分析——Gemini 2.0 Flash 自動分類...
 第三層是自動推播——Make 根據重要程度...
 第四層是展示輸出——GitHub 網站和 Looker Studio...」
````

---

### 第三段：Live Demo（3 分鐘）

這是最關鍵的部分，讓系統真實地跑起來給大家看。

**建議的 Demo 流程（選最穩定的三個功能）：**
````
Demo A：填寫表單 → 即時 Discord 通知
  操作：填寫 Google 表單送出
  展示：Discord 頻道在 10 秒內收到通知
  說明：「這背後是 Make + Webhook 的自動化」

Demo B：AI 分析的結果
  展示：打開試算表，顯示 AI 自動填入的分析欄位
  展示：打開 Notion 情報知識庫，顯示自動建立的頁面
  說明：「這是 Gemini API 分析了職缺描述後的結果」

Demo C：GitHub 展示網站
  展示：打開 Vercel 部署的網站
  展示：點擊各個頁面，展示 Looker Studio 嵌入的儀表板
  說明：「這個網站是用 AI Studio 生成 HTML，部署到 Vercel」
````

> [!WARNING]
> **Demo 的備用方案：**
> 現場網路或系統可能出問題。
> 請事先準備截圖或螢幕錄影作為備用，
> 萬一 Live Demo 失敗，立刻切換到備用方案，
> 不要花太多時間排除問題。

---

### 第四段：成果數據（1.5 分鐘）

把系統的成效量化，讓聽眾感受到具體的價值：

**建議的數據類型：**

| 數據類型 | 範例 |
|---------|------|
| 資料量 | 「本學期系統共處理了 847 筆職缺資料」|
| 時間節省 | 「每週自動化節省約 5 小時人工作業」|
| 準確率 | 「AI 分類與人工核對的一致率達 89%」|
| 覆蓋率 | 「監控了 3 個網站、2 個政府公告平台」|
| 通知數量 | 「本學期共發送 127 則 Discord 自動通知」|

**在 GitHub 網站或 Looker Studio 展示這些數字。**

## 💡 未來繼續學習的方向

根據你的興趣，以下是推薦的進階學習路徑：

---

### 路徑 A：No-Code 自動化專家

適合：喜歡設計自動化流程，不想寫太多程式碼

**下一步學習：**
- **n8n**：Make 的開源替代品，可以自己架設（免費無限制）
- **Zapier**：Make 的競爭對手，App 整合更多
- **Airtable**：比 Notion 更強大的資料庫工具
- **Power Automate**：Microsoft 生態系的自動化工具（企業常用）

---

### 路徑 B：AI 應用開發

適合：對 AI 應用有興趣，想做更複雜的 AI 系統

**下一步學習：**
- **LangChain**：建立 AI Agent，讓 AI 自主完成多步驟任務
- **Python + OpenAI API**：用程式語言操作 AI API
- **Retrieval-Augmented Generation（RAG）**：讓 AI 查詢你的私人知識庫
- **Hugging Face**：開源 AI 模型的最大平台

---

### 路徑 C：全端工程師

適合：想學會自己寫程式，不依賴 No-Code 工具

**下一步學習：**
- **JavaScript / TypeScript**：這學期的 HTML 是入門，JS 讓網頁動起來
- **React 或 Vue**：現代前端框架
- **Node.js**：用 JavaScript 寫後端
- **PostgreSQL 或 MongoDB**：真正的資料庫（不是試算表）

---

### 路徑 D：資料分析師

適合：對資料分析和視覺化有興趣

**下一步學習：**
- **Python + Pandas**：資料清洗和分析的業界標準
- **Matplotlib / Seaborn**：Python 的資料視覺化
- **SQL**：這學期的 QUERY 函數是入門，SQL 是正式版
- **Tableau**：比 Looker Studio 更強大的視覺化工具（學生可申請免費授權）

## 📬 課程結語

親愛的同學，

這門課設計的初衷，不是讓你學會幾個工具，
而是讓你相信一件事：

**你不需要成為工程師，也能用科技解決真實的問題。**

Make 的節點可以換成其他工具，
Gemini 的 API 可以換成 GPT 或 Claude，
Notion 可以換成任何資料庫，
GitHub 可以換成任何版本控制系統——

但你學會的「自動化思維」、「API 串接能力」、「資料結構觀念」，
這些是不會消失的。

下次你在工作上遇到「每週要花三小時做同一件事」的時候，
希望你會想到：**這件事可以自動化。**

然後你會打開 Make，設計一個劇本，
讓系統去做那三小時的工作，
而你，去做更重要的事。

加油。

### 常見錯誤與解決方法速查

| 錯誤情境 | 可能原因 | 解決方法 |
|---------|---------|---------|
| Make 劇本執行後沒反應 | 觸發器設定錯誤 | 確認 Watch Rows 的起始位置 |
| Discord 沒收到通知 | Webhook URL 過期或錯誤 | 重新建立 Webhook |
| Gemini 回傳格式錯誤 | AI 加了 ` ```json ` 代碼框 | 用 replace() 清除代碼框 |
| Notion API 401 錯誤 | Integration 未連結到 Database | 在 Notion 頁面「Connections」加入 Integration |
| Vercel 部署失敗 | HTML 語法錯誤 | 用瀏覽器開啟本機版確認語法 |
| ParseHub 抓不到資料 | 目標網站改版 | 重新錄製爬蟲或聯繫 ParseHub 客服 |
| VLOOKUP 回傳 #N/A | 資料有多餘空格 | 用 TRIM() 清洗後再查詢 |
| OCR 辨識失敗 | 檔案格式不符 | 確認 Content-Type 並調整 fileName |
| Google Drive 下載失敗 | 資料夾未設為公開 | 右鍵 → 共用 → 知道連結的人可以查看 |
